适配器与外观模式

适配器模式：将一个类的接口，转换成客户期望的另一个接口，适配器让原本接口不兼容的类可以合作无间。
在中国使用美国制造的笔记本电脑，需要使用一个交流电的适配器
客户使用适配器的过程如下：

  1. 客户通过目标接口调用适配器的方法对适配器发出请求
  2. 适配器使用被适配者接口把请求转换成被适配者的一个或多个调用接口
  3. 客户接收到调用的结果，但并未察觉这一切是适配器在起转换作用

外观模式(Facade-Pattern)：提供了一个统一的接口，用来访问子系统中的一群接口。外观定义了一个高层
接口，让子系统更容易使用。提供简化的接口的同时，依然把系统完整的功能暴露出来，将客户实现从子系统中解耦
适配器模式与外观模式区别：适配器意图是改变接口符合客户的期望(将接口转换成不同接口)，
而外观模式的意图是提供子系统的一个简化接口(简化接口)
适配器模式与外观模式相同点：都可以包装许多类

家庭影院观看电影，困难的方式是需要打开相关的许多子系统
将灯光调暗 -> 放下屏幕 -> 打开投影机 -> 将投影机输入切换到DVD ->将投影机设置在宽屏模式 -> 打开功放
-> 功放设置为环绕立体声 -> 打开 DVD 播放器 -> 开始播放 DVD
外观模式是通过简单的方式将这些任务写成类和方法进行封装，达到简化目的

要点
当需要使用一个现在的类而其接口并不符合你的需要时，就使用适配器
当需要简化并统一一个很大的接口或者一群复杂的接口时，使用外观
适配器改变接口以符合靠窗户的期望
外观将客户从一个复杂的子系统中解耦
实现一个外观，需要将子系统组合进外观中，然后将工作委托给子系统执行
适配器模式与装饰者模式与外观模式区别:
适配器将一个对象包装起来以改变接口；装饰者将一个对象包装起来以增加新的行为和责任；外观将一群对象
“包装”起来以简化其接口

示例
如果它走起来像鸭子，叫起来像只鸭子，那么他可能是一只包装了鸭子适配器的火鸡
